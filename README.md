This repository contains artifacts from my CS 320: Software Testing, Automation, and Quality Assurance course at Southern New Hampshire University. The included files demonstrate my ability to develop Java applications, create unit tests using JUnit, verify software requirements, and apply software testing best practices.

## Reflection

### How can I ensure that my code, program, or software is functional and secure?

I ensure my software is functional by creating unit tests that verify every requirement and validate both expected and unexpected inputs. Throughout this course, I used JUnit to test valid data, invalid data, and boundary conditions to confirm that my classes behaved correctly. I also improved security and reliability by validating user input, preventing invalid objects from being created, and ensuring exceptions were thrown when requirements were violated.

### How do I interpret user needs and incorporate them into a program?

I begin by carefully reviewing the project requirements and breaking them into individual features that can be implemented and tested. Each customer requirement became a specific method or validation rule within my code, and every requirement was verified with one or more JUnit tests. This approach helped ensure that the final application behaved exactly as the customer expected.

### How do I approach designing software?

I approach software design by first understanding the requirements and then dividing the project into small, manageable components. During this course, I developed separate Contact, Task, and Appointment services, allowing each class to focus on a single responsibility. After implementing each feature, I immediately created automated tests to verify functionality before moving on to additional development. This process helped me identify defects early, improve code quality, and reduce the amount of debugging required later in the project.
